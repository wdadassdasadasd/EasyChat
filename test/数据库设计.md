# 客户端本地数据库设计

来源：

- `weChat/src/main/db/Tables.js`
- `weChat/src/main/db/ADB.js`
- `weChat/src/main/db/ChatMessageModel.js`
- `weChat/src/main/db/ChatSessionUserModel.js`
- `weChat/src/main/db/UserSettingModel.js`

数据库：SQLite。本地数据库文件路径由 `weChat/src/main/db/ADB.js` 决定：

| 环境 | 数据库文件 |
| --- | --- |
| 开发环境 | `用户主目录/.weChattest/local.db` |
| 生产环境 | `用户主目录/.weChat/local.db` |

## 表总览

| 表名 | 表说明 |
| --- | --- |
| `chat_message` | 本地聊天消息表，保存当前登录用户相关的聊天消息 |
| `chat_session_user` | 本地会话列表表，保存左侧聊天会话数据 |
| `user_setting` | 用户本地设置表，保存邮箱、系统设置、联系人申请未读数、本地服务端口等 |

## 字段命名映射

数据库字段使用下划线命名，业务代码使用驼峰命名。映射关系定义在 `weChat/src/main/db/ADB.js` 的 `globalColumnMap`。

| 数据库字段 | 业务字段 |
| --- | --- |
| `message_id` | `messageId` |
| `session_id` | `sessionId` |
| `message_content` | `messageContent` |
| `send_user_id` | `sendUserId` |
| `send_user_nick_name` | `sendUserNickName` |
| `contact_id` | `contactId` |
| `contact_type` | `contactType` |
| `last_message` | `lastMessage` |
| `last_receive_time` | `lastReceiveTime` |
| `no_read_count` | `noReadCount` |
| `top_type` | `topType` |
| `contact_no_read` | `contactNoRead` |
| `server_port` | `serverPort` |

查询数据库后，`convertDbObj2BizObj` 会把下划线字段转换成驼峰字段。

## chat_message

表说明：本地聊天消息表。WebSocket 初始化消息和普通聊天消息会被保存到这张表。

建表来源：`Tables.js`

| 字段名 | 业务字段 | 类型 | 约束/默认值 | 字段含义 |
| --- | --- | --- | --- | --- |
| `user_id` | `userId` | `varchar` | `NOT NULL`，联合主键 | 当前登录用户 ID。本地库按用户隔离消息 |
| `message_id` | `messageId` | `bigint` | `NOT NULL DEFAULT NULL`，联合主键 | 消息 ID |
| `session_id` | `sessionId` | `varchar` | 索引 `idx_session_id` | 会话 ID，用于加载某个聊天窗口的消息 |
| `message_type` | `messageType` | `integer` | 允许为空 | 消息类型。当前代码中 `0` 用于 WebSocket 初始化消息；发送文本时前端使用 `2` |
| `message_content` | `messageContent` | `varchar` | 允许为空 | 消息内容，文本消息主要展示该字段 |
| `contact_type` | `contactType` | `integer` | 允许为空 | 联系人类型。当前代码中 `1` 表示群聊，其他值通常表示单聊 |
| `send_user_id` | `sendUserId` | `varchar` | 允许为空 | 发送人 ID。前端用它判断消息是否自己发送 |
| `send_user_nick_name` | `sendUserNickName` | `varchar` | 允许为空 | 发送人昵称，群聊展示发送人时会用到 |
| `send_time` | `sendTime` | `bigint` | 允许为空 | 消息发送时间，通常为时间戳 |
| `status` | `status` | `integer` | 允许为空 | 消息状态。当前客户端代码未明确枚举 |
| `file_size` | `fileSize` | `bigint` | 允许为空 | 文件大小，文件消息使用 |
| `file_name` | `fileName` | `varchar` | 允许为空 | 文件名，文件消息使用 |
| `file_path` | `filePath` | `varchar` | 允许为空 | 文件路径，文件消息使用 |
| `file_type` | `fileType` | `integer` | 允许为空 | 文件类型，当前客户端代码未明确枚举 |

主键：`user_id, message_id`

索引：`idx_session_id(session_id)`

相关方法：

| 方法 | 作用 |
| --- | --- |
| `saveMessage(data)` | 给消息补充当前登录用户 `userId`，然后写入 `chat_message` |
| `saveMessageBatch(chatMeassageList)` | 批量保存消息，同时更新会话未读数 |
| `selectMesssageList(query)` | 按 `sessionId` 分页查询消息 |

## chat_session_user

表说明：本地会话列表表。左侧聊天列表主要从这张表读取。

建表来源：`Tables.js`

| 字段名 | 业务字段 | 类型 | 约束/默认值 | 字段含义 |
| --- | --- | --- | --- | --- |
| `user_id` | `userId` | `varchar` | `NOT NULL DEFAULT 0`，联合主键 | 当前登录用户 ID |
| `contact_id` | `contactId` | `varchar(11)` | `NOT NULL`，联合主键 | 联系人 ID 或群组 ID |
| `contact_type` | `contactType` | `integer(1)` | 允许为空 | 联系人类型。当前代码中 `1` 表示群聊 |
| `session_id` | `sessionId` | `varchar(11)` | 允许为空 | 会话 ID，用于关联 `chat_message.session_id` |
| `status` | `status` | `integer` | `DEFAULT 1` | 会话状态：`1` 正常显示，`0` 删除/隐藏 |
| `contact_name` | `contactName` | `varchar(20)` | 允许为空 | 会话名称。单聊为联系人名称，群聊为群名称 |
| `last_message` | `lastMessage` | `varchar(500)` | 允许为空 | 最后一条消息摘要，用于左侧会话列表 |
| `last_receive_time` | `lastReceiveTime` | `bigint` | 允许为空 | 最后一条消息时间，用于会话排序 |
| `no_read_count` | `noReadCount` | `integer` | `DEFAULT 0` | 当前会话未读消息数 |
| `member_count` | `memberCount` | `integer` | 允许为空 | 群成员数量，群聊标题处会展示 |
| `top_type` | `topType` | `integer` | `DEFAULT 0` | 是否置顶：`0` 不置顶，`1` 置顶 |

主键：`user_id, contact_id`

相关方法：

| 方法 | 作用 |
| --- | --- |
| `saveOrUpdateChatSessionBatch4Init(chatSessionList)` | WebSocket 初始化时批量保存或更新会话 |
| `selectUserSessionList()` | 查询当前用户 `status = 1` 的会话列表 |
| `selectUserSessionByContactId(contactId)` | 查询当前用户与某个联系人/群的会话 |
| `updateNoReadCount({ contactId, noReadCount })` | 更新某个会话的未读消息数 |
| `delChatSession(contactId)` | 将会话 `status` 更新为 `0`，实现删除/隐藏 |
| `topChatSession(contactId, topType)` | 更新会话置顶状态 |

前端排序规则位于 `Chat.vue`：

| 排序字段 | 规则 |
| --- | --- |
| `topType` | 置顶会话排前面 |
| `lastReceiveTime` | 同样置顶状态下，最近消息时间越新越靠前 |

## user_setting

表说明：用户本地设置表。登录成功后会初始化或更新。

建表来源：`Tables.js`

| 字段名 | 业务字段 | 类型 | 约束/默认值 | 字段含义 |
| --- | --- | --- | --- | --- |
| `user_id` | `userId` | `varchar` | `NOT NULL`，主键 | 当前登录用户 ID |
| `email` | `email` | `varchar` | `NOT NULL` | 用户邮箱 |
| `sys_setting` | `sysSetting` | `varchar` | 允许为空 | 系统设置 JSON 字符串 |
| `contact_no_read` | `contactNoRead` | `integer` | 允许为空 | 联系人申请未读数量 |
| `server_port` | `serverPort` | `integer` | 允许为空 | 本地服务端口，新用户会在当前最大端口基础上递增 |

主键：`user_id`

`sys_setting` 默认结构：

| 字段 | 含义 |
| --- | --- |
| `localFileFolder` | 本地文件存储目录，默认在用户主目录下的 `.weChat/fileStorge/` |

相关方法：

| 方法 | 作用 |
| --- | --- |
| `addUserSetting(userId, email)` | 登录成功后初始化用户本地设置 |
| `updateNoReadCount(contactId, noReadCount)` | 更新联系人申请未读数 |

## 核心关系

| 关系 | 说明 |
| --- | --- |
| `chat_message.user_id` -> 当前登录用户 | 本地消息按登录用户隔离 |
| `chat_message.session_id` -> `chat_session_user.session_id` | 消息归属于某个会话 |
| `chat_session_user.user_id` -> 当前登录用户 | 会话列表按登录用户隔离 |
| `chat_session_user.contact_id` -> 联系人/群组 ID | 单聊时是好友 ID，群聊时是群 ID |
| `user_setting.user_id` -> 当前登录用户 | 每个登录用户一条本地设置 |

## 枚举汇总

以下枚举来自当前客户端代码推断，后续应与后端枚举保持一致。

| 字段 | 枚举 |
| --- | --- |
| `chat_message.message_type` | `0` WebSocket 初始化消息；`2` 文本消息 |
| `chat_message.contact_type` | `1` 群聊；其他值通常表示单聊 |
| `chat_session_user.contact_type` | `1` 群聊；其他值通常表示单聊 |
| `chat_session_user.status` | `1` 正常显示；`0` 删除/隐藏 |
| `chat_session_user.top_type` | `0` 不置顶；`1` 置顶 |

## 数据流

### 登录成功

入口：`weChat/src/main/ipc.js`

| 步骤 | 说明 |
| --- | --- |
| 1 | `store.initUserId(config.userId)` 保存当前登录用户 |
| 2 | `store.setUserData("token", config.token)` 保存 token |
| 3 | `addUserSetting(config.userId, config.email)` 初始化用户本地设置 |
| 4 | `initWs(config, e.sender)` 建立 WebSocket 连接 |

### WebSocket 初始化消息

入口：`weChat/src/main/wsClient.js`

| 步骤 | 说明 |
| --- | --- |
| 1 | 服务端推送 `messageType == 0` 的初始化消息 |
| 2 | 保存 `extendData.chatSessionList` 到 `chat_session_user` |
| 3 | 保存 `extendData.chatMessageList` 到 `chat_message` |
| 4 | 更新联系人申请未读数 |
| 5 | 通知渲染进程刷新会话列表 |

### 普通聊天消息

入口：`weChat/src/main/wsClient.js`

| 步骤 | 说明 |
| --- | --- |
| 1 | 收到普通聊天消息 |
| 2 | `saveMessageBatch([message])` 保存到 `chat_message` |
| 3 | `webContentsSender.send('receiveMessage', message)` 通知前端 |
| 4 | `Chat.vue` 判断是否为当前 `sessionId`，是则追加到 `messageList` |

### 加载左侧会话列表

| 位置 | 内容 |
| --- | --- |
| 前端事件 | `window.ipcRenderer.send('loadSessionData')` |
| 主进程监听 | `ipcMain.on('loadSessionData')` |
| 查询方法 | `selectUserSessionList()` |
| SQL | `select * from chat_session_user where user_id=? and status=1` |

### 加载聊天消息

| 位置 | 内容 |
| --- | --- |
| 前端事件 | `window.ipcRenderer.send('loadChatMessage', { sessionId, pageNo, maxMessageId })` |
| 主进程监听 | `ipcMain.on('loadChatMessage')` |
| 查询方法 | `selectMesssageList(query)` |
| 分页大小 | 每页 20 条 |
| 排序 | 按 `message_id` 升序展示 |

## 当前代码注意事项

| 位置 | 问题 |
| --- | --- |
| `ChatMessageModel.js` | `selectMesssageList` 中 SQL 是 `user_id=? and session_id=?`，但参数传入顺序疑似写成 `[sessionId, store.getUserId()]` |
| `ADB.js` | `queryCount` 当前写法 `Object.values(row[0])` 可能异常，因为 `row` 是对象不是数组 |
| `UserSettingModel.js` | 写入时疑似使用了 `sys_setting` 表名，但实际建表名是 `user_setting` |
| `UserSettingModel.js` | `updateNoReadCount(contactId, noReadCount)` 更新的是联系人申请未读数，不是聊天会话未读数 |
| `chat_session_user` | `contact_id`、`session_id` 定义为 `varchar(11)`，但后端 `session_id` 是 `varchar(32)`，后续建议统一 |
